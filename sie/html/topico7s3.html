Amazon S3 - Simple Storage Service

O Amazon S3 (Simple Storage Service) é uma solução de armazenamento de objetos da AWS que oferece escalabilidade, segurança, disponibilidade e desempenho. Ele é usado para armazenar e recuperar qualquer quantidade de dados, a qualquer momento e de qualquer lugar na web.

Buckets e Objetos

No S3, os dados são armazenados em "buckets", que são contêineres para objetos. Um bucket possui um nome global exclusivo e está vinculado a uma região específica da AWS, garantindo o controle sobre onde os dados serão armazenados fisicamente.

  • Bucket: Contêiner de objetos (arquivos e metadados).
  • Objetos: Arquivos armazenados dentro de buckets, com identificador exclusivo (chave).
  • Nome global único: O nome do bucket deve ser único em todas as contas da AWS.

Segurança no S3

O Amazon S3 oferece várias opções de segurança para proteger seus dados:

  • Política IAM: Controla o acesso ao S3 através de usuários e grupos IAM.
  • Política de Bucket: Definições para permitir ou restringir o acesso ao bucket, permitindo por exemplo a configuração de acesso público ou privado.
  • Criptografia: Os dados podem ser criptografados em repouso (usando SSE-S3, SSE-KMS, ou SSE-C) e em trânsito (com SSL/TLS).

Hospedagem de Websites Estáticos

O S3 pode ser configurado para hospedar websites estáticos, oferecendo uma solução econômica e escalável para servir conteúdo estático diretamente da nuvem. É ideal para sites que não precisam de backend dinâmico.

Versionamento no S3

O versionamento no S3 permite manter várias versões de um objeto dentro do mesmo bucket. Isso ajuda a proteger contra exclusões acidentais e a restaurar versões anteriores de arquivos.

Replicação do S3

A replicação permite que você copie objetos entre buckets em diferentes regiões (replicação entre regiões) ou dentro da mesma região (replicação dentro da região). Para habilitar a replicação, o versionamento deve estar ativado no bucket de origem.

Classes de Armazenamento do S3

O S3 oferece várias classes de armazenamento que permitem otimizar os custos de acordo com a frequência de acesso aos dados:

  • S3 Standard: Armazenamento para dados frequentemente acessados.
  • S3 Intelligent-Tiering: Otimiza custos automaticamente movendo dados entre camadas de armazenamento de acordo com o padrão de acesso.
  • S3 Standard-IA (Infrequent Access): Ideal para dados acessados com menos frequência, mas que precisam estar disponíveis rapidamente quando solicitados.
  • S3 One Zone-IA: Armazenamento de baixo custo para dados infrequentes, mas que não requerem replicação em múltiplas zonas de disponibilidade.
  • Amazon S3 Glacier: Armazenamento de longo prazo com acesso mais lento, usado para arquivamento a um custo muito baixo.
  • Amazon S3 Glacier Deep Archive: A opção mais econômica para arquivamento de dados que raramente são acessados.

Família Snow e OpsHub

A Família Snow oferece dispositivos físicos (como o Snowball e Snowmobile) que permitem a importação e exportação de grandes volumes de dados para o S3. É útil quando a conectividade de rede é limitada ou muito lenta.

O OpsHub é um aplicativo de desktop que facilita o gerenciamento dos dispositivos da Família Snow, permitindo a transferência de dados de forma eficiente.

Amazon S3 Storage Gateway

O Storage Gateway é uma solução híbrida que integra o armazenamento local com o Amazon S3, permitindo que empresas utilizem o S3 para backup, arquivamento e recuperação de desastres, enquanto mantêm seus dados acessíveis localmente.
